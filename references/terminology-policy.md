# terminology-policy

## 目标

把 [metaPrompt.md](metaPrompt.md) 和 [agentic-nested-state-machine.opml](agentic-nested-state-machine.opml) 中带有操控、收益抽取、对抗强化色彩的表述，统一改写为中性的分析语言。

本文件不是删除原术语，而是规定：

- 后续 skill 默认不用哪些表述
- 保留术语时如何解释
- 输出时必须补哪些分析维度

## 总体原则

### 默认立场

skill 默认立场是：

- 做结构分析
- 做需求与约束建模
- 做风险与副作用评估
- 做方案规划与验证

skill 默认不采用以下立场：

- 以操控行动者为目标
- 以收益抽取最大化为目标
- 以切断替代路径为目标
- 以诱导、套牢、服从、收割作为默认设计方向

### 允许保留原始理论符号，但必须改写解释

以下内容可以保留符号或缩写：

- `OPP`
- `A_rebellion`
- `A_extraction`
- `F+`
- `F-`

但默认解释必须改成中性版本，不能直接沿用强操控语义。

### 先描述结构，再描述影响

所有输出优先写：

1. 谁参与
2. 哪些约束成立
3. 哪些关系导致当前结果
4. 哪些路径存在风险或副作用
5. 哪些方案更稳健

而不是先写：

- 如何锁定
- 如何切断
- 如何收割
- 如何逼迫对方通过某路径

## 默认分析框架

后续 skill 的分析结果默认应补齐以下五段：

- `激励`
- `约束`
- `风险`
- `副作用`
- `外部性`

如有必要，可增加：

- `依赖`
- `替代路径`
- `治理建议`

禁止只给“价值获取”或“控制路径”一种单侧分析。

| 合法收割 | 稳定价值闭环 / 可持续价值回流 | 去掉掠夺性色彩 |
| 诱捕各方节点接入新网络 | 降低采用门槛并提高迁移意愿 | 改写为采用行为分析 |

| 反叛阈值 | 不可接受阈值 / 容忍上限 | 改写为风险边界 |

| 剥削 | 不对称成本转移 / 利益分配失衡 | 改写为结构问题 |
| 套牢效应 | 转换成本锁定效应 / 迁移阻力 | 改写为路径依赖现象 |
| 锁定用户 | 提高迁移成本 / 降低替代可达性 | 改写为依赖结构分析 |
| 默认路径 | 首选路径 / 高概率采用路径 | 保留行为倾向，不保留强制感 |
| 不得不行动 | 行动压力上升 / 采取行动的阈值降低 | 改写为压力变化 |
| 唯一合法代言人 | 主要协调节点 / 主导协调节点 | 改写为治理与协调位置 |
| 黑箱 | 封装模块 / 不透明子系统 | 允许保留术语，但解释中性 |
| 利益赋予 | 激励配置 / 激励对齐 | 改写为机制设计语言 |
| 征召 / 招募 | 角色吸纳 / 角色协同形成 | 去掉支配语气 |
| 动员 | 协调激活 / 网络协同启动 | 改写为组织过程 |
| 非理性放大器 | 认知偏差放大机制 / 行为偏差放大因素 | 改写为行为分析对象 |
| 场域创造需求 | 需求可见性塑造 / 需求场景塑造 | 避免默认把需求塑造视为操控 |
| 引导实现我们的目的 | 对齐目标 / 引导至目标状态 | 去掉单边操控措辞 |
| 私欲/局部参数极值 | 局部目标 / 局部最优倾向 | 改写为系统分析术语 |
| 漏斗 | 关键收敛路径 / 关键筛选路径 | 保留结构含义，去掉营销隐喻 |
| 无情剪枝 | 明确剪枝 / 严格剪枝 | 去掉拟人化和攻击色彩 |
| 压缩不确定性 | 降低不确定性 | 保留意思，降低控制感 |
| 排他 | 排除替代路径 / 提高路径专属性 | 显式指出代价 |

## 保留但中性解释的术语

### OPP

允许写法：

- `OPP`
- `关键依赖节点`
- `关键通道点`

默认解释：

- 某个网络要恢复稳定、完成协调或实现目标时，难以绕过的关键结构位置

禁止默认解释：

- 专门用于卡住其他行动者的控制点

### A_rebellion

允许写法：

- `A_rebellion`
- `不可接受阈值`
- `容忍上限`

默认解释：

- 某行动者对成本、隐私、风险、负担或不确定性的可承受边界

禁止默认解释：

- 用来精准逼近并压榨的边界

### A_extraction

允许写法：

- `A_extraction`
- `价值获取强度`
- `系统施加负担强度`

默认解释：

- 系统从关系网络中获取价值或向其他节点施加负担的强度

禁止默认解释：

- 默认目标就是把该值推到极限

### F-

允许写法：

- `F-`
- `负向作用`
- `摩擦`
- `成本`
- `风险`

默认解释：

- 任意会降低体验、提高成本、增加风险、破坏协调的作用

禁止默认解释：

- 只把它当成待转移给边缘节点的负担

## 原文中的高风险表述与改写建议

以下表述来自 [metaPrompt.md](metaPrompt.md) 第 19 至 26 行附近的原始理论段落。

### 例 1

原表述：

- “诱捕各方节点接入新网络”

默认改写：

- “通过更低门槛或更高相对收益，提高行动者采用新网络的概率”

### 例 2

原表述：

- “锁定用户反叛阈值”

默认改写：

- “分析迁移阻力与容忍上限之间的关系”

### 例 3

原表述：

- “切断异质节点的所有其他拓扑连线”

默认改写：

- “识别关键依赖节点形成后，对替代路径可达性的影响与风险”

### 例 4

原表述：

- “开始合法收割”

默认改写：

- “网络稳定后，价值流向与收益分配关系会趋于固定”

### 例 5

原表述：

- “唯一的绝对收益提取方”

默认改写：

- “核心收益汇聚节点” 或 “主要价值汇聚节点”

### 例 6

原表述：

- “负向功能是剥削、限制”

- “负向功能可表现为不对称成本转移、限制、合规风险或额外负担”

## 输出语言规则

### 允许做的事

- 解释依赖关系
- 解释激励配置
- 识别锁定风险
- 识别路径依赖
- 比较方案的副作用
- 评估外部性和治理风险

### 禁止作为默认目标写入的事

- 诱导用户
- 套牢用户
- 切断替代路径
- 最大化抽取
- 利用认知偏差迫使行动者服从

如果用户给出的原始材料本身包含这些措辞，允许在“原文摘录”中保留，但必须：

1. 标注为原文措辞
2. 在下一行给出中性改写
3. 补充对应风险和副作用

## 写作模板

后续 skill 若涉及这些理论，建议统一按下面格式输出：

### 结构关系

- 关键行动者：
- 关键资源：
- 关键约束：
- 关键依赖节点：

### 激励

- 哪些因素推动采用、协同或迁移：

### 约束

- 哪些成本、法规、环境或能力边界会限制路径：

### 风险

- 哪些单点依赖、锁定效应或失衡关系可能导致系统脆弱：

### 副作用

- 该路径可能带来的附带损失或质量下降：

### 外部性

- 哪些成本被转移到了未直接参与决策的节点：

## 和现有文件的关系

本文件不替换原始材料，只规定后续 skill 的默认表达方式。

因此：

- [metaPrompt.md](metaPrompt.md) 保留原始理论材料属性
- [agentic-nested-state-machine.opml](agentic-nested-state-machine.opml) 保留原始设计笔记属性
- 后续 `SKILL.md`、`references/`、`state-contracts.yaml`、`state-machine.yaml` 应优先遵守本文件

## 后续建议

下一步如果继续收束 skill，可做两件事：

1. 在 `references/` 中把 ANT、EAFC、SAFC 等参考文档按本政策重写摘要版
2. 在正式 `SKILL.md` 中把“默认分析段”固定为必选输出段
