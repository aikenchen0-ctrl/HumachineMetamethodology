### 2. Low-priority reference refresh

Rule:

- if a quick scan finds no active boundary break, keep them deferred

### 3. Runtime/provider/governance elaboration

Boundary work may be reopened only if at least one of these becomes true:

1. a downstream consumer cannot use the current shell without a missing distinction
2. the maintained source changes in a way that breaks the current compression
3. retrieval, generation, evaluation, or pluginization semantics drift apart again
4. orchestration / runtime / provider / governance behavior is explicitly requested as real operational scope

## Final Decision Table

| Situation | Decision |
| --- | --- |
| Source, shell, contracts, and release truth all still agree | stop expanding, consolidate |
| A file is old but semantically harmless | defer |
| A wording issue can mislead source authority or default-core boundaries | fix now |
| A new real consumer forces a missing distinction | reopen narrowly |
| Team only wants “a cleaner architecture” without pressure | do not reopen |
