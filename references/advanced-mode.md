# advanced-mode

## 作用

这份文档把 `metaPrompt v2` 主链中需要更深系统建模的部分，明确映射为 skill 的高级产物层。

默认情况下，主文件已经包含：

- 问题定义；
- `E/A/F/Cond` 建模；
- 状态、约束与理想解；
- 检索与证据补全；
- 候选解生成；
- 评估与插件沉淀。

只有当用户明确要求更深的图结构建模、流分析、SAFC、USIT32 级深推、系统级解决方案集，才启用本模式。

本模式的激活不等于：

- 每个深度产物都必须在同一轮全部生成
- 一旦进入高级模式就替代基础主链
- 只要提到“系统”二字就自动进入深层建模

## 激活条件

满足任一条件即可激活：

- 用户明确要求进入主链第 `7` 章到第 `10` 章的深层建模扩展
- 用户明确要求 `ANT / EAFC / SAFC / TRIZ / USIT32 / 流分析 / 资源图 / 行动者网络图`
- 当前问题已经完成基础 `constraints-matrix`，但仍需要更深的系统结构建模才能继续
- 用户明确要求“不要只做结构化，要继续做系统级求解”

## 高级模式顺序

读取方式：

- 把下面的顺序视为高级模式的推荐展开路径
- 不要把它误读成“激活后必须全量逐项跑完”的默认义务
- 当前轮次应只展开真正服务于关键阻滞点或关键矛盾的深度产物
- 如果一个更早的深度产物已经足以恢复主链，不要为了形式完整继续扩张

### 资源扫描与资源图

产物：

- `resource-graph.json`

动作证据：

- `search-notes.json`

可选规则产物：

- `resource-priority-policy.json`

目标：

- 遍历当前系统中的可用资源
- 把扫描动作、优先级规则、资源图结构拆开
- 把资源写成显式结构，而不是散落在自然语言里

### 行动者网络图

产物：

- `actor-network.json`

目标：

- 回到宏观层，明确谁与谁发生作用
- 明确关键连接关系、控制点、评价路径与即将异动

### 功能物场与流分析

产物：

- `function-field-model.json`
- `flow-analysis.json`

目标：

- 明确功能关系
- 明确物质流、能量流、信息流
- 指出起点、终点、阻滞点、泄漏点、过载点

### SAFC 模型

产物：

- `safc-model.json`

目标：

- 对关键系统组件做排序
- 压缩关键实体、属性、功能和隐式条件

### 矛盾分析与 USIT32 深推

产物：

- `contradiction-analysis.json`

目标：

- 明确参数矛盾、功能矛盾、条件矛盾
- 指出当前最优先求解的矛盾
- 在需要时进入 `USIT32` 或相邻方法的更深展开

### 解决方案集

产物：

- `solution-set.json`

目标：

- 汇总候选解
- 给出适用条件、主要收益、风险、副作用和实施门槛

### 高级模式结束前的批判辩证复盘

默认结束产物：

- `critique-report.json`

边界说明：

- 当高级模式被正式声明为完成时，`critique-report.json` 是默认结束产物
- 但在 source-aligned skill self-review 场景中，`critique-report.json` 也可以作为独立复盘产物出现，而不自动意味着整条高级模式已完成

### 高级模式可选扩展：多屏幕分析

可选产物：

- `multi-screen-analysis.json`

适用时机：

- 用户需要超系统 / 当前系统 / 子系统 / 反系统的演化视角

## 和基础模式的关系

高级模式不是替代基础模式，而是依赖基础模式。

如果没有：

- `problem-brief.json`
- `constraints-matrix.json`
- `branch-plan.json`

高级模式会失焦。

如果基础模式仍未稳定，不要用高级模式去掩盖：

- 术语未澄清
- 约束未建模
- 分支未成形
- 评估未收束

## 结束条件

如果高级模式被激活，且当前轮要被声明为“高级模式完成”，则在 finish 前至少应满足：

- 已产出至少一个深度系统模型
- 已显式指出关键矛盾或关键阻滞点
- 已生成 `solution-set.json`
- 已生成 `critique-report.json`

如果这些条件不满足，正确做法通常是：

- 回到基础主链补强
- 或只保留当前已经真正需要的深度产物

而不是继续机械扩写更多高级模式 artifact。
